Overview of OSI Layer 3

The Network Layer is responsible for routing packets and determining the best path across networks. It uses logical addressing schemes such as IP and includes protocols like ICMP and IPSec.

  • IP (Internet Protocol) – connectionless, no acknowledgements
  • ICMP – used for diagnostics and control
  • IPSec – used for encrypted communications

IP Addressing

IP addresses are 32-bit logical identifiers used to distinguish hosts on a network. They allow segmentation into subnets for better security, performance, and manageability.

Example: 192.168.10.15

Each IP address consists of four octets (4 × 8 bits = 32 bits).

Binary and Decimal

Humans use decimal (base 10), while computers use binary (base 2).

Decimal Example: 236 → 2 hundreds, 3 tens, 6 ones

Binary Example: 236 = 11101100

Bit place values: 128 64 32 16 8 4 2 1

Subnet Masks & Logical Separation

Subnet masks define the boundary between network and host portions of an IP address. They are also 32 bits and can be written in dotted decimal or slash notation.

Example: 192.168.10.15 / 255.255.255.0 = 192.168.10.15/24

Binary form:
IP: 11000000.10101000.00001010.00001111
Mask: 11111111.11111111.11111111.00000000

1's in the mask = network portion; 0's = host portion.

Useful IP Tools

  • ipconfig – Windows
  • ifconfig – Linux/Mac
  • show ip interface brief – Cisco

Types of IP Traffic

  • Unicast: One-to-one
  • Broadcast: One-to-all (within subnet only; routers drop broadcast)
  • Multicast: One-to-many (only to subscribed hosts)
